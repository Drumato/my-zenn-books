# はじめに

## 挨拶

こんにちは。Drumatoです。  
この本はLinuxなどで標準採用されている、  
**ELF(Executable and Linkable Format)** というバイナリ形式を理解しつつ、  
実際にバイナリ解析を楽しんでもらおう、という趣旨のもと執筆されました。  

ELFを理解することで、  
一般的には次のような恩恵が得られると思います。

- **ツールチェインの出力結果が読めるようになる**
  - 動的リンク・静的リンクされたバイナリを読めるようになります
  - **オブジェクトファイルやコアダンプ、共有ライブラリもELFに則っています**
- ELFを扱うソフトウェアが書けるようになる
  - **ELF解析器やリンカ等を作る人にとっては必須の知識です**
  - 自作エミュレータ等に興味がある人にとっては特に楽しいかも
    - やっぱり既存ツールチェインの生成する実行ファイルを読みたいですよね
- 他のバイナリ形式を理解する前段階としても良い
  - **ELFはかなり単純な構造になっているので、入門者におすすめです**
  - PE32+など、別のバイナリ形式はELFよりよっぽど複雑ですが、バイナリ解析のコツは活かせます
- 単にELFの構造について話せる人はあまり多くない
  - **私はELFが好きなので、お友達が増えて私が嬉しいです(何が"一般的な恩恵"なのか)**

## この本でわかること

- ELFファイルの構造
- Rustでバイナリパーサを記述するときのコツ
- この本の後、より深く理解するためにやること
  - バイナリ解析がやりたい人
  - ツールチェイン自作をやりたい人
  - 高機能なELF解析器を作りたい人

## この本でわからないこと

- 各属性値がどのような意味を持ち、リンカ等がどのように解釈するのか
- ELFセクションの中身の深い話
  - 例えば、x64分岐命令の形式みたいな、機械語の話とか
  - 動的セクションにはこんな情報があって、こういう使われ方をしてるとか
- LinuxカーネルがELFローダをどう実装しているかとか
  - 気になる人は`execve(2)`とかを読むといいと思います
- アセンブラがどのようにオブジェクトファイルを生成しているのか
  - `gcc -o`等を注意深く観察すると色々わかります
- readelfのような高機能ELF解析ツールの作り方
  - この本で作るパーサは、主にバイナリ表現をRustの構造体へ読み込むところまでを扱います
  - セクション内容の詳細表示やリンカ視点の解釈までは深く扱いません

## 環境

Linux + Rust環境であれば、特に問題なく動きます。  
ユーザ空間プログラムなので、VMなりWSLなりで普通に動作します。  

筆者の環境を貼っておきます。

```shell-session
$ uname -a
Linux debian 4.19.0-17-amd64 #1 SMP Debian 4.19.194-3 (2021-07-18) x86_64 GNU/Linux
$ cargo version
cargo 1.54.0 (5ae8d74b3 2021-06-22)
```

## 実装

この本で実装するELFパーサはデモ水準であるため、実用性はないので注意してください。  

## 注意点

この本では、実際に必要な実装のうち大部分を省いています。  
実用的なELFパーサを作るためには、

- 各フィールドで許されていない範囲のバイトを見つけたらエラー
- 実装していない形式はパニックではなくエラー
- 使いやすさを高めるための仕組み

などなど(これ以外にもあります)、いくつかの考慮事項があると思います。  
が、今回はあくまでも"最小限の理解"を目指すので、これらの多くを無視します。  
ご自身で実際に作るときは、ソフトウェアの堅牢性に注意してください。  

また、私はELFの専門家ではありません。  
自分の作りたいものを作るために必要最低限な情報だけ拾ってきたので、  
すごい詳しい仕様とかにはあまり明るくありません。  

Rustのもっと良い書き方があるとか、  
説明が間違っているみたいなことがあったらぜひスクラップに書いていただけると。  
というか、スクラップに自分の書いたELFパーサを載っけて盛り上がってくださいね!
