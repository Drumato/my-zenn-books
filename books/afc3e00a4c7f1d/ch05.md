# セクションヘッダテーブルと文字列テーブル

ELFヘッダパーサが完成すると、  
セクションヘッダテーブルのパーサを作れるようになります。  

ここで、以下2つの前提知識について復習しておきます。  

- **セクション名はセクションヘッダテーブルに直接埋め込まれない**
  - これはセクションヘッダを固定長で実現するために必要
  - ということでELFの文字列テーブル形式の理解が必要
- **セクションの内容はセクション種別によって異なる**
  - 例えばsymtabならシンボルエントリの配列になっている
  - progbitsならベタに機械語が並んでいる

## セクションヘッダ

まずはセクションヘッダの中身です。  
ELFヘッダと同じように一部のみ取り上げます。  

|    名前    | サイズ(バイト) |                  説明                   |
| :--------: | :---------: | :--------------------------------------------: |
|  name idx  |      4      |              セクション名のインデックス               |
|    type    |      4      |       PROGBITS/NOTEなどセクションの種類        |
|   flags    |      8      |         マージを許したり書き込みを許したり         |
|    addr    |      8      |  セクションをどの仮想アドレスに対応付けるか   |
|   offset   |      8      |          セクション内容の開始オフセット          |
|    size    |      8      |        セクション内容のファイル上でのサイズ        |
|    link    |      4      |    セクション種別によって異なる使われ方をする    |
|    info    |      4      |    セクション種別によって異なる使われ方をする    |
| addr_align |      8      |        セクション内容のアラインメント        |
| entry_size |      8      | セクション内容がテーブル形式の場合に使われる |

このように、セクション名は間接的に読み込むようになっています。  
そして、そのセクション名が書き込まれているセクションが存在し、  
そのセクションインデックスはELFヘッダに書き込まれています。  

つまり、以下の手順で **"セクション名が格納されているセクションの内容"** を取得する必要があります。  

- ELFヘッダを読む
- セクションヘッダテーブルを読む
- `header.sht_string_table_index` からセクション名テーブルを取ってくる
- それを解析して使える形にする
- `shdr.name_idx` を渡してセクション名を取り出す

### 文字列テーブル形式

とその前に、肝心の文字列テーブル形式について説明しておきますが、  
大体こんな感覚を持ってくれれば良いと思います。  

```text
section_names = "\x00" section_name "\x00" section_name "\x00" ...
section_name = take_while(printable_ascii)
```

`\x00`を`?`と表すと、  
`?.text?.symtab?.strtab` のようになるわけです。  

重要なのは、セクションヘッダの `name_idx` は文字列テーブル内の **バイトオフセット** だという点です。  
例えば `name_idx == 1` なら、文字列テーブルの1バイト目からNUL文字までを読む、という意味になります。  
「1番目の文字列を読む」という意味ではありません。

## セクションヘッダパーサ

それではまずはセクションヘッダパーサを作ります。  
基本的なやり方はch03と同じです。  
テストコードは省略します。  

```rust
// src/parser/section.rs
fn parse_64bit_section_header(raw: &[u8]) -> IResult<&[u8], SectionHeader64> {
    let (r, name_idx) = le_u32(raw)?;
    let (r, ty) = parse_section_type(r)?;
    let (r, flags) = le_u64(r)?;
    let (r, addr) = le_u64(r)?;
    let (r, offset) = le_u64(r)?;
    let (r, size) = le_u64(r)?;
    let (r, link) = le_u32(r)?;
    let (r, info) = le_u32(r)?;
    let (r, addr_align) = le_u64(r)?;
    let (r, entry_size) = le_u64(r)?;

    Ok((
        r,
        SectionHeader64 {
            name_idx,
            ty,
            flags,
            addr,
            offset,
            size,
            link,
            info,
            addr_align,
            entry_size,
        },
    ))
}
```

```rust
// src/parser/section.rs
pub fn parse_64bit_section_header_table<'a>(
    entries: usize,
) -> impl Fn(&'a [u8]) -> IResult<&'a [u8], Vec<SectionHeader64>> {
    move |raw: &'a [u8]| count(parse_64bit_section_header, entries)(raw)
}
```

### 文字列テーブルエントリパーサ

ここでは文字列テーブルのエントリを一つ解析する関数を作ります。  

```rust
// src/parser/string_table.rs
pub fn parse_string_table_entry(raw: &[u8]) -> IResult<&[u8], String> {
    terminated(parse_ascii_string, tag(b"\x00"))(raw)
}

fn parse_ascii_string(raw: &[u8]) -> IResult<&[u8], String> {
    map(
        take_while(|c: u8| c.is_ascii_graphic()),
        |raw_name: &[u8]| String::from_utf8(raw_name.to_vec()).unwrap().to_string(),
    )(raw)
}
```

ここでは説明を簡単にするため、セクション名をASCIIの表示可能文字だけとして扱っています。  
実用的なパーサにする場合は、文字列テーブルの範囲外参照やUTF-8として解釈できないバイト列もエラーとして扱う必要があります。

## ELFファイルパーサ

まだプログラムヘッダテーブルの解析部分を作っていませんが、  
先にELF全体を解析する関数を作ってしまいましょう。  

```rust
// src/parser/file.rs
pub fn parse_64bit_elf(raw: &[u8]) -> IResult<&[u8], Elf64> {
    let (_, header) = parse_64bit_elf_header(raw)?;
    let (_, sht) = parse_64bit_section_header_table(header.sht_entries as usize)(
        &raw[header.sht_offset as usize..],
    )?;
    let raw_section_names_offset = sht[header.sht_string_table_index as usize].offset;
    let raw_section_names = &raw[raw_section_names_offset as usize..];
    let sections = sht
        .into_iter()
        .map(|header| Section64 {
            name: parse_string_table_entry(&raw_section_names[header.name_idx as usize..])
                .unwrap()
                .1,
            header,
        })
        .collect::<Vec<Section64>>();

    Ok((&raw[raw.len()..], Elf64 { header, sections }))
}
```

ここで注意すべき点として、 **`name_idx` と `sht` のインデックスは別物** ということです。  

例えば `sht[1]` が `.text` セクションだったとして、  
`sht[1].name_idx == 1` になるとは限りません。  
`name_idx` は文字列テーブル内のバイト位置なので、`sht` の何番目のセクションかとは対応しません。  

また、この実装では `raw[header.sht_offset as usize..]` や `sht[header.sht_string_table_index as usize]` のように直接スライスしています。  
学習用に短く書いていますが、不正なELFを渡すとpanicする可能性があります。  
実用的にする場合は、オフセットやサイズがファイル範囲内に収まっているかを確認してから読むようにしてください。
