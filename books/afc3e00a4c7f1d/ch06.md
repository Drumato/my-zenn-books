# プログラムヘッダテーブル

リンカが生成する、プログラムヘッダテーブルのパーサも書いていきましょう。  

readelfなどを使うとセクションとセグメントの対応付けが見れたりしますが、  
今回は作りません。  
実用的なツールを作る場合は必要になると思います。

やり方は全く同じです。  

- ELFヘッダを読む
- `header.pht_offset` から `sizeof(Elf64_Phdr) * header.pht_entries` 分読んでデシリアライズ

再配置可能オブジェクトファイルなど、プログラムヘッダテーブルがまだ付与されていないバイナリも存在します。

## パーサ

まず、プログラムヘッダを表す構造体とセグメント種別を用意します。  
ここでも全ての種別を網羅せず、よく見る値だけを扱います。

```rust
// src/elf/program_header.rs
#[derive(PartialEq, Eq, PartialOrd, Ord, Debug)]
pub enum SegmentType {
    Null,
    Load,
    Dynamic,
    Interp,
    Note,
    ShLib,
    Phdr,
    Tls,
    Unknown,
}

impl From<u32> for SegmentType {
    fn from(v: u32) -> SegmentType {
        match v {
            0 => SegmentType::Null,
            1 => SegmentType::Load,
            2 => SegmentType::Dynamic,
            3 => SegmentType::Interp,
            4 => SegmentType::Note,
            5 => SegmentType::ShLib,
            6 => SegmentType::Phdr,
            7 => SegmentType::Tls,
            _ => SegmentType::Unknown,
        }
    }
}

#[derive(PartialEq, Eq, PartialOrd, Ord, Debug)]
pub struct ProgramHeader64 {
    pub ty: SegmentType,
    pub flags: u32,
    pub offset: u64,
    pub virtual_addr: u64,
    pub physical_addr: u64,
    pub size_in_file: u64,
    pub size_in_mem: u64,
    pub align: u64,
}
```

```rust
// src/parser/program_header.rs
pub fn parse_64bit_program_header_table<'a>(
    entries: usize,
) -> impl Fn(&'a [u8]) -> IResult<&'a [u8], Vec<ProgramHeader64>> {
    move |raw: &'a [u8]| count(parse_64bit_program_header, entries)(raw)
}

fn parse_segment_type(raw: &[u8]) -> IResult<&[u8], SegmentType> {
    map(le_u32, |v| SegmentType::from(v))(raw)
}

fn parse_64bit_program_header(raw: &[u8]) -> IResult<&[u8], ProgramHeader64> {
    let (r, ty) = parse_segment_type(raw)?;
    let (r, flags) = le_u32(r)?;
    let (r, offset) = le_u64(r)?;
    let (r, virtual_addr) = le_u64(r)?;
    let (r, physical_addr) = le_u64(r)?;
    let (r, size_in_file) = le_u64(r)?;
    let (r, size_in_mem) = le_u64(r)?;
    let (r, align) = le_u64(r)?;

    Ok((
        r,
        ProgramHeader64 {
            ty,
            flags,
            offset,
            virtual_addr,
            physical_addr,
            size_in_file,
            size_in_mem,
            align,
        },
    ))
}
```

`Elf64` にプログラムヘッダテーブルも持たせます。

```rust
// src/elf/file.rs
pub struct Elf64 {
    pub header: ElfHeader64,
    pub sections: Vec<Section64>,
    pub program_headers: Vec<ProgramHeader64>,
}
```

そして、ELF全体のパーサで `header.pht_offset` からプログラムヘッダテーブルを読みます。  
再配置可能オブジェクトファイルのように `pht_entries == 0` の場合は、空の `Vec` が返ります。

```rust
// src/parser/file.rs
pub fn parse_64bit_elf(raw: &[u8]) -> IResult<&[u8], Elf64> {
    let (_, header) = parse_64bit_elf_header(raw)?;
    let (_, sht) = parse_64bit_section_header_table(header.sht_entries as usize)(
        &raw[header.sht_offset as usize..],
    )?;
    let (_, program_headers) = parse_64bit_program_header_table(header.pht_entries as usize)(
        &raw[header.pht_offset as usize..],
    )?;

    let raw_section_names_offset = sht[header.sht_string_table_index as usize].offset;
    let raw_section_names = &raw[raw_section_names_offset as usize..];
    let sections = sht
        .into_iter()
        .map(|header| Section64 {
            name: parse_string_table_entry(&raw_section_names[header.name_idx as usize..])
                .unwrap()
                .1,
            header,
        })
        .collect::<Vec<Section64>>();

    Ok((
        &raw[raw.len()..],
        Elf64 {
            header,
            sections,
            program_headers,
        },
    ))
}
```
