# シンボルテーブルパーサへの挑戦

ここまでの内容をもとに、  
**シンボルテーブルを格納するセクションの中身を解析する** 、ということができるようになっています。  

やり方(とそれに必要な事前知識)を説明します。  

- やり方
  - セクションヘッダテーブルを反復処理します
  - `section type == SHT_SYMTAB` で絞り込みます
    - `sht.iter().filter(|shdr| shdr.ty == SectionType::SymTab)` みたいな感じ
    - 動的シンボルも見たい場合は `SHT_DYNSYM` も同じ要領で扱えます
  - セクション内容を `shdr.offset` をもとにシークします
    - `raw[shdr.offset..(shdr.offset + shdr.size)]` のようになると思います
    - シンボルテーブルの各エントリは `Elf64_Sym` という構造体になっています(`/usr/include/elf.h` を確認)
    - シンボルテーブルのエントリ数は `shdr.size / shdr.entry_size` で計算可能です
    - これでSHT/PHTと同様に `nom::multi::count()` を使って作れます
  - 次にシンボル名ですが、これも同じようにシンボル名テーブルが必要です
    - シンボル名テーブルのセクションインデックスは `symtab_section_header.link` が表しています
- 注意点
  - シンボルテーブルにも同様にNULLシンボルが存在します

完成させると自分が今まで定義した関数のシンボルとかが見れるようになるはずです。
